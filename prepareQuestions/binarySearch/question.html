Mihir is working hard in competitive programming as he aspires to join Uber , Whenever Mihir gets some free time, he starts solving Questions. Today he's got t free minutes to solve questions.So , Mihir found a list of n questions and for each question he estimates the time he is going to need to solve it. Let's number the questions by integers from 1 to n. Mihir needs ai minutes to solve the i-th question.

Mihir decided to choose a random question with number i and solve the questions one by one, starting from this question. In other words, he will first solve question number i, then question number i + 1, then question number i + 2 and so on. He continues the process until he either runs out of the free time or finishes solving the n-th question. Mihir solves each question up to the end, that is, he doesn't start solving the question if he doesn't have enough free time to finish solving it.

Print the maximum number of question Mihir can solve.

The first line contains two integers n and t (1 ≤ n ≤ 105; 1 ≤ t ≤ 109) — the number of questions and the number of free minutes Mihir got. The second line contains a sequence of n integers a1, a2, ..., an (1 ≤ ai ≤ 104), where number ai shows the number of minutes that he needs to solve the i-th question.
